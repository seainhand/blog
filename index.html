AJAX那些事（二）jQuery中的AJAX

jQuery中AJAX的快捷API

jQuery为AJAX封装了许多可以快捷使用的API，如果请求不是很复杂，使用快捷API是一个明智的选择

下面将介绍几个常用的快捷API

jQuery.get & jQuery.post

两个方法除了非别以 GET , POST 方法发送请求外，在参数，API功能和使用方法上完全相同。

$.get(url,[data],[callback],[type])
$.get(url,[data],[callback],[type])

url 表示当前请求所要发送的位置

data 表示当前请求所要携带的数据，必须为一个key-value形式的object对象

callback 表示请求成功之后的回调方法，请求发生错误无法使用

type 指定当前请求返回数据的编码格式json，htm等，基本不用

一个简单的ajax如下

$.get('/test'),{a:1},function(data,callbacktype,jqXHR){
    console.log(arguments);
}):

callback的方法有三个参数data,callbacktype,jqXHR

data 表示请求成功后的响应数据

callbacktype 表示一个表示响应状态成功还是失败的字符串，由于只有成功才回调，所以一般没用

jqXHR jquery封装的xhr对象，一般将其当做普通的xhr对象

一般的callback方法只用data一个参数

jQuery(selector).load

load方法用于加载页面片段

$(selector).load(url,[data],[callback])

selector 表示加载请求返回的内容选择器

data 表示当前请求所要携带的数据，必须为一个key-value形式的object对象

callback 表示请求成功之后的回调方法

简单的例子

$(#containter).load('/test',{a:1},function(){
    console.log('返回成功');
})

即向’/test’位置发送一个key，value分别为a和1的数据对象，返回一个页面片段加载到id为container的html代码中，并打印“返回成功”。

jQuery.getScript

getScript方法用来动态加载脚本

$.getScript(url,[callback])

url 表示脚本的位置

callback 表示请求成功之后的回调方法

jQuery中AJAX的核心$.ajax

$.ajax是jQuery对AJAX的底层封装，上面几个快捷API实际上都是调用这个方法
$.ajax有两种调用方式

$.ajax(url,[settings])
$.ajax(settings)

因为第一种比较机智，一般使用第一种，所以下文就使用第一种了

url 表示当前请求所要发送的位置

settings 表示一个object对象，指明了当前请求的行为方式如提交方法，回调函数，请求数据等

下面将讲解settings中的各种参数

处理响应结果的回调函数

如success，error，statueCode，complete

$.ajax('/test',{

    success:function(data){
        console.log('成功');
    },
    //success参数表示当请求成功后的回调函数
    //参数data表示返回的数据

    error:function(jqXHR,textstatus,err){
        console.log('失败');
    },
    //error参数表示当请求失败后的回调函数
    //jqXHR:jquery封装的xhr对象，加强版xhr.
    //textstatus:请求完成状态，返回失败即为error
    //err:返回的异常信息

    complete:function(jqXHR,textstatus){
        console.log('statusCode: %d,statusText:%s',jqXHR.status,jqXHR.statusText);
        console.log('statusText:%s',textstatus);
    },
    //complete无论请求失败或者成功都回调
    //jqXHR:jquery封装的xhr对象，加强版xhr.
    //textstatus:请求完成状态，返回失败即为error

    statusCode:{
        '403':function(jqXHR,textstatus,err){
            console.log('失败');
        },
        '400':function(jqXHR,textstatus,err){
            console.log('失败');
        }
    }
    //statusCode属性的成员值是一个object对象，对象的每一项表示一个返回的http状态码，并且对应回调函数
});

请求数据属性

如data,processData,contentType,traditional

$.ajax('/test',{
    data: JSON.stringify({
        a:1,
        b:2,
        c:[3,4],
    }),
    processData:false,
    contentType:'application/json',
    traditonal:true,
    //data:表示当前请求所要携带的数据
    //processData:表示是否根据响应的情况将data转换形式
    //contentType:表示data的内容类型，此例中由于是json字符串所以data要使用JSON.stringify方法转换data
    //traditonal:表示是否用传统方式编码，一般在data中包含数组时使用
});

请求数据属性

如dataType,dataFilter

$.ajax('/test',{
    dataType: 'json',
    dataFilter:function(data,type){
        var d = JSON.parse(data);
        return d;
    }
    //dataType:表示当前请求返回数据的目标类型
    //dataFilter:表示一个将data进行变为目标类型预处理的函数,其参数其中data表示返回的原始文本数据，type表示目标类型。

});

其他属性

请求类型:type默认为GET，可以为GET、POST、PUT、DELETE等

同步异步: async，默认为true，表示异步

是否缓存: cache, 默认为true

前置处理：beforeSend

设置请求头: headers

以上简单介绍了jQuery中的AJAX，如有必要更多将于后续篇章写出

AJAX那些事（一）基础

我们为什么要使用AJAX

想象这样一个注册用户的场景

当我们在注册用户的时候可能是这样一个表单

如果我们的注册用户名不符合规范，需要将全部input填写完并提交注册后才会提醒，但是此时我们已经浪费的大量的时间,这样的用户体验十分差。

我们需要的是在填写完用户名之后立即提醒用户名是否可以使用，如下图

AJAX正是完成此需求的最常用的手段,

什么是AJAX

AJAX 是一个简称（缩写），
分别由以下四个单词的首字母组成

Asynchronous         浏览器使用的异步通信，实现页面局部刷新(操作DOM树)

Javascript               使用的编程语言

And

Xml                          数据通信的承载方式，实际上很少使用XML

即使用javascript的编程语言，使用XML(JSON)等数据承载方式，实现异步通信，浏览器页面局部刷新。

我们如何使用AJAX

使用一个AJAX的流程为

  1. 创建承载访问和返回内容的对象，即AJAX的核心XMLHttpRequest对象
  • 告诉这个对象你要发请求到哪去，发的这个请求包含什么
  • 告诉这个对象你发成功了，返回了之后你要做些什么
  • 发送这个对象

分别用代码演示

创建XMLHttpRequest对象

//支持IE7+、以及其他主流浏览器
var xhr = XMLHttpRequest();
//支持全浏览器
var xhr = null;
if(window.XMLHttpRequest){
    xhr = new XMLHttpRequest();
} else if (window.ActiveXObject) {
    try{
        xhr = new ActiveXObject('Msxml2.XMLHTTP');
    } catch (e) {
        try{
            xhr = new ActiveXObject('Microsoft.XMLHTTP');
        }
    }
}

要注意的是XMLHttpRequest对象分level1和level2

老版本的XMLHttpRequest level有着一些缺点

  • 只支持文本数据的传送，无法用来读取和上传二进制文件。
  • 传送和接收数据时，没有进度信息，只能提示有没有完成。
  • 受到”同域限制”（Same Origin Policy），只能向同一域名的服务器请求数据。

新版本的XMLHttpRequest level2新增了很多功能

  • 可以设置HTTP请求的时限。
  • 可以使用FormData对象管理表单数据。
  • 可以上传文件。
  • 可以请求不同域名下的数据（跨域请求）。
  • 可以获取服务器端的二进制数据。
  • 可以获得数据传输的进度信息。
    　　
    当使用level2时请注意浏览器的兼容问题

设定请求包含内容

open方法进行初始化操作，初始请求的一些信息，使用open方法后，XMLHttpRequest对象变为open状态，才可以进行发送。

open方法包含五个参数

xhr.open(
    //请求提交的方式，一般用GET方法,或POST方法
    'GET', 

    //请求的目标，即URL，使用GET方法是将传递的参数写入URL中
    'http://seainhand.github.io/test?user=lixiang', 

    //确定请求的为同步或者异步，异步为true,同步为false，正常情况下为true
    true, 

    //用户名，当服务器需要用户名密码验证的时候使用
    'username', 

    //密码，当服务器需要用户名密码验证的时候使用
    'password', 
);

setRequestHeader方法给请求加入头部信息

//给请求头加入Content-Type信息
xhr.setRequestHeader('Content-Type','application/json');

//给请求头加入自定义的user信息
xhr.setRequestHeader('user','lixiang');

设定请求完成后进行的动作

XMLHttpRequest有一个名为readystate的属性，它表示XMLHttpRequest对象的五种状态，即

  • 0:未初始化
  • 1 : 连接建立，请求发出
  • 2 : 服务器返回相应
  • 3 : 交互（处理相应数据）
  • 4 : 完成，数据可交付客户端使用

只有当readystate等于4,并且需要http的状态码为200时，我们才有必要处理我们的逻辑代码，即

if (xhr.readyState === 4 && xhr.status == 200) {

    //获取返回的文本
    console.log(xhr.responseText);

    //将字符串转化为JSON对象
    JSON.parse(xhr.responseText);
    eval(xhr.responseText);

    //获取返回的XML        
    console.log(xhr.responseXML);

    //获取响应的头部信息
    console.log(xhr.getAllResponseHeaders());

    //获取响应的某些头部信息
    console.log(xhr.getResponseHeader("Content-Type"));

    //获取响应的状态码
    console.log(xhr.status);

    //获取响应的状态文本
    console.log(xhr.statusText);
}

发送请求

当XMLHttpRequest对象为open状态时可以使用send方法发送请求

//当提交方式为POST方法时，将参数写入send方法中，使用urlencoded模式，即用&链接的key=value值
xhr.send('user=lixiang&age=20&sex=male'); 

//JSON模式,HTTP请求只能发送文本，所以使用JSON.stringfy方法进行序列化
xhr.send(JSON.stringfy({
    user : lixiang,
    age : 20,
    sex : male
}));

这样我们就完成了一个AJAX过程
