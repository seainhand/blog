Universal Link（通用链接）完全实现！

断更11个月，并不是死了，只是一不小心参加了工作。(￣▽￣)”

在 ios9 和 ios9+ 中使用 Universal Link 可以做到点击短信或者邮件中的链接直接跳到app，或者从微信直接跳到app并且不需要进入微信的白名单这样的效果，这样的操作用户体验还是极好极好的呀。

##两个域名

首先你需要拥有两个域名，注意并不是子域名，但是类似于 a.domain.com 和 b.domain.com是可以的，a.domain.com/a 和 a.domain.com/b 是不可以的。

##开发者中心配置
然后你需要在开发者中心做配置。具体是这样：开发者中心->Identitiers->App IDs->需要支持的应用App ID，在Application Services列表里有Associated Domains一条，把它变为Enabled就可以了，如下图

##IOS注册

然后, 你必须在Xcode的capabilities里添加你的APP域名，并且使用“applinks: ”前置它，如图

这样APP会在第一次打开的时候从你配置的域名下，下载一个名叫apple-app-site-association的JSON文件，这个文件将会配置跳转允许的路径表，如下图

##JSON文件

这个名叫apple-app-site-association的文件使用JSON格式，如下

{
    "applinks": {
        "apps": [],
        "details": [
            {
                "appID": "lixiang.com.apple.shuai",
                "paths": [ "/src/WebBundle/", "/src/*"]
            }
        ]
    }
}

1.这其中”apps” 只要有并且定义数组就可以了 并不需要填充

2.”appId” 是由”开发者的TeamID” + “.” + “APP的bundle”，组成的比如说TeamID为”lixiang”，应用的Bundle为”com.apple.shuai”，那么”appId”为”lixiang.com.apple.shuai”

3.”paths”为该域名下哪些路径可以链接跳转到APP，可以使用通配符”*”

####请注意：

1.apple-app-site-association文件没有后缀名，文件全名即为”apple-app-site-association”

2.apple-app-site-association必须在跳转域名的根目录下，即在浏览器输入”https://b.domain.cn/apple-app-site-association"即能下载apple-app-site-association文件

##支持ssl

接下来你需要使你准备的两个域名中的其中一个域名支持ssl，这个域名并不是你微信网页的域名，而是而是跳转链接的域名，也是”IOS注册”这一步中注册的的域名，当你完成上一步之后，第一次手机打开你的APP，都会自动前往这个注册的域名使用ssl的链接下载这个文件。

##完成检验

Apple有检测通用链接是否可用的网站:”https://search.developer.apple.com/appsearch-validation-tool/"，输入你的跳转网址，当"Universal Link”一栏中为passed就可以了，然而当在开发模式的时候，会出现下图问题

在这种情况下，其实通用连接也可以使用了，只是相关APP在开发模式下，并没有上线而已
